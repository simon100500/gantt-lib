## [0.20.0] - 2026-03-18

### Критические изменения
- **Удалены:** пропсы `weekends` и `workdays` - заменены единым массивом `customDays`
- **Новое:** проп `customDays?: CustomDayConfig[]`, где `CustomDayConfig = { date: Date; type: 'weekend' | 'workday' }`
- Миграция: замените `weekends={[date1, date2]}` на `customDays={[{ date: date1, type: 'weekend' }, { date: date2, type: 'weekend' }]}`

### Рефакторинг
- Унификация API кастомных выходных/рабочих дней в единый массив `customDays` с явной типизацией
- Добавлен интерфейс `CustomDayConfig` и утилита `createCustomDayPredicate` с O(1) lookup на основе Set
- Обновлены все компоненты (GanttChart, TaskList, TaskListRow, Calendar, DatePicker) для использования нового API
- Сохранён проп `isWeekend?: (date: Date) => boolean` для кастомной логики предиката

### Возможности
- Добавлен порядок приоритетов для кастомных дней: workday > weekend > базовый предикат > по умолчанию

### Исправления
- Обновлена демо-страница для использования нового API `customDays`

### Тесты
- Добавлено 8 новых модульных тестов для `createCustomDayPredicate`, покрывающих все сценарии
