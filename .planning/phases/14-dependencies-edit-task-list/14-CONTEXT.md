# Phase 14: dependencies-edit-task-list - Context

**Gathered:** 2026-03-03
**Status:** Ready for planning

<domain>
## Phase Boundary

Добавить столбец «Связи» в компонент TaskList, позволяющий создавать, просматривать и удалять связи между задачами. Пользователь выбирает глобальный тип связи в заголовке столбца, нажимает «+» на строке и кликает по строке-предшественнику. Связи отображаются чипами. Редактирование лагов — отдельная фича (out of scope).

</domain>

<decisions>
## Implementation Decisions

### Отображение связей в ячейке
- Чипы формата `ОН(2)`, `НН(5)` — тип + порядковый номер предшественника (позиция строки в таблице)
- При переполнении: показывать 2–3 чипа, остальные скрыть за кнопкой «+N ещё»
- Лаг в чипе не отображать — только тип и номер
- Кнопка «+» для добавления новой связи — в конце списка чипов в ячейке

### Выбор глобального типа связи
- Переключатель типа находится в заголовке столбца: «Связи [ОН ▾]»
- Клик по заголовку открывает выпадающее меню с вариантами: ОН, НН, ОО, НО (только аббревиатуры)
- Активный тип отображается в скобках рядом со словом «Связи»
- Тип по умолчанию при открытии таск листа: ОН

### Пикер задачи-предшественника
- После нажатия «+» — таск лист переходит в режим выбора предшественника
- В режиме выбора: строки в столбце «Связи» подсвечиваются при наведении (hover)
- Клик по любой строке (кроме текущей) — добавляет связь с выбранным глобальным типом
- Отмена режима: Escape или клик за пределами таск листа
- Цикл (A → B → A): блокировать добавление + показать всплывающее сообщение об ошибке

### Удаление связей
- На чипе при hover появляется крестик ×; клик — удаляет связь немедленно, без подтверждения
- Кнопка «+N ещё» открывает попапер со полным списком всех связей задачи, откуда тоже можно удалить любую
- Пропс `disableDependencyEditing` (по аналогии с `disableTaskNameEditing`) — скрывает «+», × и меню типа; столбец становится read-only

### Claude's Discretion
- Позиционирование попапера «+N ещё» (использовать существующий Popover компонент)
- Точная анимация/стиль чипов и крестика
- Стиль выделения строк в режиме выбора предшественника

</decisions>

## Existing Code Insights

### Reusable Assets
- `Popover` (packages/gantt-lib/src/components/ui/Popover.tsx): уже используется в DatePicker — переиспользовать для меню типа и для попапера «+N ещё»
- `dependencyUtils.ts` → `validateDependencies()` с `detectCycle` — уже готов для проверки циклов при добавлении связи
- `TaskListRow.tsx` — сюда добавляется новая ячейка «Связи»; уже есть паттерн с `onClick` + `stopPropagation`
- `TaskList.tsx` — сюда добавляется заголовок столбца с переключателем типа; уже есть паттерн `disableTaskNameEditing`

### Established Patterns
- CSS-классы с префиксом `gantt-tl-*` — продолжать этот паттерн для новых элементов столбца
- `onTaskChange` callback — через него передавать изменения `task.dependencies[]` наверх (как уже делается для имени и дат)
- `Popover` с `portal={true}` — использовать для попапера типа и расширенного списка связей (чтобы не обрезался overflow таск листа)
- Состояние активного типа (`activeLinkType`) — хранить в `TaskList` (не в GanttChart), передавать вниз в `TaskListRow`
- Режим выбора предшественника (`selectingPredecessorFor`) — хранить в `TaskList`, передавать в строки

### Integration Points
- `Task.dependencies?: TaskDependency[]` — уже существует в types/index.ts; `TaskDependency = { taskId, type: LinkType, lag?: number }`
- `LinkType = 'FS' | 'SS' | 'FF' | 'SF'` — маппинг на отображение: FS↔ОН, SS↔НН, FF↔ОО, SF↔НО
- `TaskList` → `TaskListRow`: через `props` передать `activeLinkType`, `selectingPredecessorFor`, `allTasks` (для пикера), `onAddDependency`, `onRemoveDependency`
- `GanttChart` уже управляет зависимостями (`task.dependencies`) — новый столбец изменяет тот же массив через `onTaskChange`

</code_context>

<specifics>
## Specific Ideas

- Аббревиатуры — русские: ОН (окончание→начало), НН (начало→начало), ОО (окончание→окончание), НО (начало→окончание)
- Порядковый номер в чипе — визуальный индекс строки (rowIndex + 1), не task.id
- В режиме выбора предшественника — текущая задача недоступна для выбора (нельзя создать связь с собой)

</specifics>
