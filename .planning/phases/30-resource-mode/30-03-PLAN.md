<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| pointer events -> move callback | Untrusted browser mouse coordinates become consumer-visible resource move requests. |
| callback -> consumer authorization | Consumer decides whether a resource reassignment is allowed before updating state. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-30-03-01 | Tampering | `useResourceItemDrag` | mitigate | Gate drag start when `readonly` or `item.locked` is true and never call `onResourceItemMove` for those cases. |
| T-30-03-02 | Denial of Service | document-level drag listeners | mitigate | Attach listeners only during active drag and remove them on mouseup/unmount; use RAF-throttled preview updates following existing hook patterns. |
| T-30-03-03 | Elevation of Privilege | resource reassignment | transfer | Emit `fromResourceId` and `toResourceId`; authorization is consumer-owned and must be validated before the consumer updates `resources`. |
| T-30-03-04 | Tampering | drop outside resource rows | mitigate | Resolve target resource from current Y coordinate and cancel without callback when no row is hit. |
</threat_model>
