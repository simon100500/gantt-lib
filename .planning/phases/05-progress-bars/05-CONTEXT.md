# Phase 5: Progress Bars - Context

**Gathered:** 2026-02-20
**Status:** Ready for planning

## Phase Boundary

Отображение прогресса выполнения задач (0-100%) на полосах задач диаграммы Ганта. Прогресс только визуальный — пользователь не может его изменять через UI. Изменение прогресса происходит только программно через props.

## Implementation Decisions

### Визуальное отображение
- Горизонтальное заполнение фона полосы задачи слева направо
- Цвет прогресса: более тёмный полупрозрачный оттенок от основного цвета задачи
- Текст процента НЕ отображается (только визуальное заполнение)
- Прогресс 0% не отображается (скрыт)
- Прогресс 100% отображается заполненной полосой

### Взаимодействие
- Пользователь НЕ может изменять прогресс через UI
- Drag-маркеры для изменения прогресса НЕ нужны
- onProgressChange колбэк НЕ нужен

### Данные и события
- Формат: `number` от 0 до 100 (дробные значения разрешены, например 45.5)
- Свойство опциональное: `progress?: number` (undefined = нет прогресса)
- Глобальная опция для цвета прогресса: `progressColor`, `progressSelectedColor`
- Колбэки для изменения прогресса не нужны

### Граничные случаи
- Дробные значения округляются до ближайшего целого (45.7 → 46)
- Значения < 0 трактуются как 0, > 100 как 100 (clamp в диапазон)
- Параметр `accepted: boolean` для принятых задач (опциональный)

### Стилизация выполненных задач
- 100% (выполнена, но не принята): жёлтый цвет прогресса
- 100% + accepted=true (принята): зелёный цвет прогресса

### Claude's Discretion
- Точная реализация визуализации более тёмного полупрозрачного цвета
- Сглаживание краёв прогресс-бара
- Позиционирование и z-index прогресс-бара относительно текста задачи

## Specific Ideas

Нет конкретных референсов — открыт для стандартных подходов к отображению прогресса на Gantt диаграммах

## Deferred Ideas

- Зависимости между задачами — отдельная фаза
- Вехи (milestones) — отдельная фаза
