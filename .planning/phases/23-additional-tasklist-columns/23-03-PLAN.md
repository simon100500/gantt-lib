    В `packages/gantt-lib/src/components/TaskList/TaskListRow.tsx` добавить state:

    ```typescript
    const [editingCustomColumnId, setEditingCustomColumnId] = useState<string | null>(null);
    ```

    Для каждой custom column собирать реальный row context:
    ```typescript
    const isEditing = editingCustomColumnId === column.id;

    const openEditor = () => {
      if (column.editor) setEditingCustomColumnId(column.id);
    };

    const closeEditor = () => {
      if (editingCustomColumnId === column.id) setEditingCustomColumnId(null);
    };

    const updateTask = (patch: Partial<TTask>) => {
      onTasksChange?.([{ ...task, ...patch } as TTask]);
      setEditingCustomColumnId(null);
    };
    ```

    Правила поведения:
    - Если у колонки нет `editor`, клик по cell не должен менять `editingCustomColumnId`.
    - Если у колонки есть `editor`, body cell становится clickable trigger и вызывает `openEditor()`.
    - В режиме `isEditing=true` body cell рендерит `column.editor(context)` вместо `column.renderCell(context)`.
    - Editor container обязан делать `event.stopPropagation()` на click/mousedown, чтобы не ломать row selection и внешние handlers.
    - Сохранять данные только через `updateTask`, который делает exact merge `{ ...task, ...patch }` и вызывает существующий `onTasksChange`.
    - Отдельный callback уровня колонки не добавлять.
    - Одновременно в одной строке может быть открыт только один custom editor (`editingCustomColumnId` single-value state).

    Добавить DOM-маркеры:
    - `data-custom-column-id="{column.id}"`
    - `data-custom-column-editing="true|false"`
    - `data-custom-column-editor="{column.id}"` на editor wrapper
