# Phase 19: hierachy - Context

**Gathered:** 2026-03-10
**Status:** Ready for planning

<domain>
## Phase Boundary

Иерархия задач — родительские и дочерние задачи с одним уровнем вложенности. Родительские задачи автоматически агрегируют даты и прогресс от детей, поддерживают сворачивание, визуально отличаются и могут участвовать в зависимостях как обычные задачи.
</domain>

<decisions>
## Implementation Decisions

### Уровни вложенности
- Один уровень: родитель → дети
- Нет глубокой вложенности (упрощает UI и логику)

### Визуальное представление
- **Список задач**: дочерние задачи имеют отступ вправо относительно родителя
- **Сворачивание**: родительскую задачу можно сворачивать/разворачивать (кнопка +/-)
- **Стиль родителя**: отличается от обычной задачи (другой тип полосы на шкале времени, другой вид в списке)
- **Шкала времени**: родительская полоса выделяется визуально (другой стиль/цвет)

### Даты родительской задачи
- Автоматически рассчитываются от детей:
  - `startDate = min(children.startDate)`
  - `endDate = max(children.endDate)`
- Обновляются в реальном времени при перемещении детей

### Прогресс родительской задачи
- Взвешенная сумма по длительности детей
- Каждая задача учитывается пропорционально своей длительности
- Автоматический пересчёт при изменении прогресса детей

### Каскадное поведение при перетаскивании
- **Перемещение родителя**: все дети сдвигаются на ту же дельту (как dependency cascade)
- **Перемещение ребёнка**: даты родителя автоматически обновляются (расширяются/сужаются)
- Двусторонняя связь между родителем и детьми

### Создание иерархии
- Всплывающие кнопки "понизить/повысить" уровень задачи
- Позволяет быстро менять иерархию без drag & drop

### Зависимости
- Родительская задача может иметь зависимости (predecessor/successor)
- Работает как обычная задача в плане связей
- Зависимости можно добавлять как на родителя, так и на детей

### Удаление задач
- При удалении родителя — удаляются все дети (каскадное удаление)
- При удалении ребёнка — родитель обновляется (даты пересчитываются)

### Claude's Discretion
- Точное визуальное оформление родительской полосы (цвет, тень, границы)
- Позиция кнопок сворачивания и повышения/понижения
- Детали анимации сворачивания/разворачивания
- Обработка конфликтов при перемещении (например, ребёнок за пределами видимости)

</decisions>

<specifics>
## Specific Ideas

- Всплывающие кнопки "понизить/повысить" появляются при наведении или выборе задачи
- Кнопка сворачивания (+/-) находится рядом с номером задачи в списке
- Родительская задача может быть пустой (без детей) — например, для будущего группирования
- При сворачивании дети скрываются из списка и с временной шкалы

</specifics>

<code_context>
## Existing Code Insights

### Reusable Assets
- **TaskList** компонент — уже есть отрисовка строк, можно добавить отступы для детей
- **TaskRow** компонент — можно расширить для родительского стиля
- **TaskBar** компонент — можно добавить стиль для родительской полосы
- **useState/useCallback** паттерны — для управления сворачиванием
- **Cascade engine** (Phase 7) — готовая логика каскадного перемещения задач

### Established Patterns
- **CSS переменные**: `gantt-*` префикс для всех стилей
- **React.memo** для оптимизации рендеринга строк
- **Callback паттерн**: `onChange`, `onAdd`, `onDelete` для коммуникации с родительским компонентом
- **Каскад через overridePosition**: механизм из Phase 7 для перемещения связанных задач

### Integration Points
- **Task type**: добавить поле `parentId?: string` для связи с родителем
- **TaskRow**: добавить проп `isParent: boolean`, `isChild: boolean`, `level: number`
- **GanttChart**: добавить пропс `onParentToggle?`, `onLevelChange?`
- **TaskList**: рендерить детей с отступом под родителем
- **Зависимости**: родитель может участвовать в `dependencies` как обычная задача

### Новые типы
```typescript
interface Task {
  // ... existing fields
  parentId?: string;  // ID родительской задачи (опционально)
  isParent?: boolean; // true если есть дети (вычисляемое или хранимое)
}
```

</code_context>
