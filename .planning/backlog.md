  ---                                                                                                                                                                                    Моё мнение: что стоит добавить
                                                                                                                                                                                       
  1. Кастомные рендеры (приоритет: высокий)

  Текущая реализация жёстко завязана на свою структуру Task. Добавить:
  renderBar?: (task: Task) => ReactNode
  renderLabel?: (task: Task) => ReactNode
  renderTooltip?: (task: Task) => ReactNode
  Это даст гибкость для кастомных визуализаций без форка библиотеки.

  2. Теги (Tag system) (приоритет: средний)

  Из прототипа видны цветные "пилюльки" рядом с названием задачи (Сек. А, Эт. 1, и т.д.).
  tags?: Tag[]  // { label, color, background }
  Полезно для строительных проектов, где нужно показывать контекст (секция, этаж, бригада).

  3. Tooltip при hover (приоритет: средний)
  <!-- В прототипе есть красивый tooltip с информацией. Сейчас в библиотеке нет всплывающих подсказок — полезно для UX. -->

  4. Легенда (приоритет: низкий)
  <!-- В прототипе внизу есть легенда с цветами работ. Простая, но полезная штука для понимания что есть что. -->

  5. onBarHover/onBarClick события (приоритет: средний)
  Сейчас только onChange при drag. Добавить отдельные события для интерактива — откроет возможности для интеграций.

  6. showProgress toggle (приоритет: низкий)
  Прогресс сейчас всегда показывается. Опциональность может быть полезна.
