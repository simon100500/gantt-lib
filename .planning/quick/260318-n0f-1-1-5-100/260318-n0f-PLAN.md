<objective>
Исправить вёрстку текста длительности на узких полосах задач (1 день)

Когда длительность задачи 1 день, полоса слишком узкая для текста "1 д". Белый шрифт сливается с фоном. Необходимо выносить длительность за пределы полосы (аналогично прогрессу), если она не помещается.

**Почему это важно:**
Пользователь должен видеть длительность любой задачи, независимо от её размера. Текст не должен сливаться с фоном.

**Результат:**
- Длительность отображается внутри полосы, если хватает места
- Длительность выносится вправо за пределы полосы, если не помещается
- Сохраняется порядок: длительность → прогресс (если вынесен) → название задачи
</objective>

## Текущая логика (строки 271-316 в TaskRow.tsx)

```typescript
// Определение вмещается ли прогресс внутри полосы
const estimatedTextWidth = durationDays >= 10 ? 76 : 62;
const showProgressInside = progressWidth > 0 && displayWidth > estimatedTextWidth;

// Рендер длительности (всегда внутри полосы)
<span className="gantt-tr-taskDuration">
  {isParent ? getChildCountLabel(childCount) : `${durationDays} д`}
</span>

// Прогресс выносится, если не помещается
{progressWidth > 0 && !showProgressInside && (
  <span className="gantt-tr-externalProgress">
    {progressWidth}%
  </span>
)}
```

**Проблема:** Длительность всегда рендерится внутри полосы (строки 309-311), нет проверки на `displayWidth`.

## Ширина полосы

- `dayWidth` — ширина одного дня в пикселях (передаётся как prop)
- Для 1 дня: `displayWidth = dayWidth` (обычно ~40-50px)
- Текст "1 д" занимает ~20-25px шрифтом 0.875rem
- С padding (0.5rem с каждой стороны = ~16px) мало места для текста

## Аналогичное решение для прогресса

Прогресс уже выносится за пределы полосы, когда `!showProgressInside` (строки 356-360).
Нужно применить аналогичную логику для длительности.
